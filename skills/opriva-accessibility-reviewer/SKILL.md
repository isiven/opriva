# Opriva Accessibility Reviewer

## When To Use

Use this skill for Opriva UI changes, including forms, tables, filters, buttons, modals, drawers, import flows, dashboards, alerts, warnings, and empty states.

## Review Checklist

- Check visual contrast for text, badges, borders, warnings, and disabled states.
- Confirm interactive elements have visible `focus-visible` states.
- Verify keyboard navigation works in a logical order.
- Ensure inputs, selects, buttons, filters, and icon controls have clear labels or accessible names.
- Use basic semantic structure for headings, buttons, forms, tables, dialogs, and alerts.
- Make disabled states understandable and not dependent on color alone.
- Ensure errors and warnings are readable, specific, and connected to the relevant control or row.
- Check text size, spacing, and density for readability in enterprise data-heavy screens.
- Confirm tables have clear headers and scannable column meaning.
- Verify modals and drawers have logical focus entry, close controls, and focus return.
- Avoid relying only on color to communicate status, risk, severity, or validation state.

## Required Output Format

1. Accessibility findings
2. Required changes
3. Keyboard / focus risks
4. Contrast / readability risks
5. Remaining risks
