# Opriva MSP / Integrator Operations Reviewer

## When To Use

Use this skill when evaluating MSP / Integrator workflows, Nextcom-style operations, client renewal portfolios, commercial ownership, distributor dependencies, margins, support coverage sold to clients, or renewal pipeline execution.

## Opriva-Specific Rules

- MSP / Integrator is external-client and commercial-renewal oriented.
- Required mental model: Client + Brand + Product + Distributor/Provider + Value + Margin + Owner + Action.
- Do not replace Client with Department in MSP views.
- Do not use Internal IT approval/budget language where MSP needs quote, proposal, distributor, owner, margin, and follow-up language.
- Tasks are operational entities linked to records, not a simple Next Action field.
- Imported records should appear as first-class local records in Licenses, Hardware, Contracts, Companies / Clients, Assets & Renewals, drawers, Reports, and Dashboard where safe.
- Backend is required later for multi-user workflows, persistent pipeline, audit, documents, and alerts.

## Review Checklist

- Client record clarity and account ownership.
- Commercial owner, seller workflow, and follow-up accountability.
- Renewals by client, brand, product, distributor/provider, owner, and status.
- Annual value, vendor cost, and margin visibility.
- Distributor quote blockers and proposal readiness.
- Support coverage sold to clients and contract relationships.
- Opportunity and renewal pipeline usefulness.
- Sales-to-operations handoff through tasks, documents, activity, and relationships.
- Dashboard/Reports usefulness for MSP leaders.

## Required Output Format

1. Executive summary
2. What is working
3. Risks / gaps
4. Recommended changes
5. Backend implications
6. MSP / Integrator implications
7. Internal IT implications
8. Suggested Codex implementation prompt, if action is needed
